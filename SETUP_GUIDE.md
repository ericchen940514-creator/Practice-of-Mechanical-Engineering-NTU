## 3. 驗證 Arduino Nano 是否正常

這步用內建的 Blink 範例，確認板子通電正常、燒錄成功。

### 3-1. 用 USB 連接 Arduino Nano 到電腦

### 3-2. 開啟 Arduino IDE，設定板子

- 上方選單 → **工具（Tools）**
- **板子（Board）** → 搜尋並選 **Arduino Nano**
- **處理器（Processor）** → 先選 **ATmega328P（Old Bootloader）**

> 若燒錄失敗，再改成 **ATmega328P**（不加 Old Bootloader）試試，兩種 bootloader 版本都很常見。

### 3-3. 選擇正確的 COM 埠

- **工具 → 連接埠（Port）** → 選擇帶有「USB」或「CH340」字樣的 COM 埠
- 若不確定是哪個，先拔掉 Arduino，看哪個 COM 埠消失，那個就是

### 3-4. 上傳 Blink 範例

- **檔案（File）→ 範例（Examples）→ 01.Basics → Blink**
- 點選上傳按鈕（→ 箭頭圖示）
- 等待下方顯示「**上傳完成（Done uploading）**」

### 3-5. 確認結果

Arduino Nano 板子上的 **L 燈（位於板子中央）** 應該開始每秒閃爍一次。

✅ **閃爍正常 → 板子沒問題，繼續下一步**
❌ **上傳失敗 → 改換處理器版本（3-2 步驟），或換一條 USB 線**

## 4. HC-05 藍牙模組接線

### 4-1. 認識 HC-05 的腳位

HC-05 模組通常有 6 支腳，從左到右（依模組正面標示）：

```
[ STATE | RXD | TXD | GND | VCC | EN ]
```

| 腳位 | 說明 |
|------|------|
| STATE | 連線狀態指示（可不接） |
| RXD | 接收資料（模組的輸入，3.3V 邏輯）|
| TXD | 傳送資料（模組的輸出）|
| GND | 接地 |
| VCC | 電源 3.3V ~ 5V |
| EN | 進入 AT 指令模式用（一般使用時不接）|

### 4-2. 接線圖

本專題的資料流向是**雙向的**：電腦發送指令 → HC-05 → Arduino，Arduino 也會將高度感測器數值回傳給電腦。因此 D10 和 D11 **都必須接**。

> ⚠️ HC-05 的 RXD 只接受 3.3V 邏輯電位，而 Arduino TX（D11）輸出為 5V，**必須透過電阻分壓降壓後再接**，否則可能損壞 HC-05 模組。

#### 所需接線（4 條）

```
Arduino Nano                    HC-05
─────────────                   ─────
    5V  ──────────────────────  VCC
   GND  ──────────────────────  GND
   D10  ──────────────────────  TXD   （接收電腦指令）
   D11  ── 1kΩ ── 節點 ── ──  RXD   （回傳高度資料）
                   │
                  2kΩ
                   │
                  GND
```

| 連接方式 | 從哪裡 | 到哪裡 |
|----------|--------|--------|
| 母對母杜邦線 | Arduino **5V** | HC-05 **VCC** |
| 母對母杜邦線 | Arduino **GND** | HC-05 **GND** |
| 母對母杜邦線 | Arduino **D10** | HC-05 **TXD** |
| 杜邦線 + 電阻分壓 | Arduino **D11** | HC-05 **RXD**（需降壓）|

#### D11 → HC-05 RXD 降壓接法說明

```
D11 (5V) ─── 1kΩ ─── 節點 ─── 2kΩ ─── GND
                       ↑
                  HC-05 RXD
                 ≈ 3.33V ✅
```

5V × 2kΩ ÷ (1kΩ + 2kΩ) ≈ **3.33V**，安全範圍內。電阻部分需借助麵包板串接。

### 4-3. 接線完成後確認

- HC-05 模組上有一顆 **LED 燈**
- 通電後應快速閃爍（約每秒 2 次）→ 代表等待配對中，**接線正確**
- 若完全不亮 → 檢查 VCC / GND 是否接對

### 4-4. （選用）更改藍牙模組名稱與密碼

HC-05 出廠預設名稱為 `HC-05`、配對密碼為 `1234`。若需要更改，需透過 **AT 指令模式**設定，步驟如下：

#### 步驟一：上傳 AT 中繼程式

先拔掉 HC-05 的 D10 接線，然後上傳以下程式（讓電腦透過 Arduino 跟 HC-05 溝通）：

```cpp
#include <SoftwareSerial.h>
SoftwareSerial BTSerial(10, 11);

void setup() {
  Serial.begin(38400);    // AT 模式固定用 38400
  BTSerial.begin(38400);
  Serial.println("AT 模式已就緒，請輸入指令...");
}

void loop() {
  if (BTSerial.available()) Serial.write(BTSerial.read());
  if (Serial.available())   BTSerial.write(Serial.read());
}
```

#### 步驟二：讓 HC-05 進入 AT 模式

1. **先拔掉 HC-05 的電源（VCC 線）**
2. 用手按住 HC-05 模組上的小按鈕（EN 按鈕）不放
3. 接回 VCC，繼續按住 2 秒後放開
4. HC-05 的 LED 改為**緩慢閃爍（約每 2 秒一次）**→ 代表已進入 AT 模式

> 若 HC-05 沒有按鈕，可將 EN 腳接到 3.3V 後再通電，效果相同。

#### 步驟三：用 Serial Monitor 發送指令

- Arduino IDE → **工具 → 序列埠監控視窗（Serial Monitor）**
- 右下角設定：鮑率 **38400**、換行選 **Both NL & CR**
- 輸入 `AT` 後按 Enter，若回應 `OK` 代表 AT 模式正常

#### 常用 AT 指令

| 指令 | 說明 | 範例 |
|------|------|------|
| `AT` | 測試連線 | 回應 `OK` |
| `AT+NAME=新名稱` | 更改藍牙名稱 | `AT+NAME=Drone_BT` |
| `AT+PSWD=新密碼` | 更改配對密碼 | `AT+PSWD=9999` |
| `AT+NAME?` | 查詢目前名稱 | 回應 `+NAME:HC-05` |
| `AT+PSWD?` | 查詢目前密碼 | 回應 `+PSWD:1234` |

**範例操作流程：**

```
AT
OK

AT+NAME?
+NAME:HC-05
OK

AT+NAME=Drone_BT
OKName:Drone_BT

AT+PSWD=9999
OK
```

#### 步驟四：設定完成後恢復正常模式

斷電再重新接上 VCC（**不要按** EN 按鈕），LED 恢復快速閃爍即代表已回到正常模式。

接著繼續 Step 5，將正式程式燒錄回 Arduino。

## 5. 燒錄正式程式到 Arduino

**先拔掉 HC-05 的 D10 和 D11 接線**，再連接 USB 燒錄。

> 原因：燒錄時若串列腳位有其他裝置連接，可能導致燒錄失敗或衝突。

### 5-0. 安裝 VL53L0X 函式庫

`sketch_althold_v3` 使用 VL53L0X 雷射測距感測器，燒錄前需先安裝對應函式庫：

1. Arduino IDE → **工具 → 管理函式庫（Library Manager）**
2. 搜尋 `VL53L0X`
3. 找到 **「VL53L0X by Pololu」** → 點選**安裝**

### 5-1. 開啟 `sketch_althold_v3.ino` 並上傳

- Arduino IDE → **檔案 → 開啟** → 選擇 `sketch_althold_v3/sketch_althold_v3.ino`
- 確認板子、處理器、COM 埠設定與第 3 步相同
- 點選上傳，等待「**上傳完成（Done uploading）**」

上傳完成後，**再把 D10、D11 的接線接回去**。

## 6. 電腦配對 HC-05

### 6-1. 開啟 Windows 藍牙設定

- 開始選單 → 設定 → **藍牙與裝置** → 開啟藍牙 → **新增裝置**

### 6-2. 配對 HC-05

- 掃描到名稱為 **HC-05**（或 **H-C-2010**）的裝置 → 點選
- 輸入配對碼：**1234**（預設值，若失敗試 **0000**）
- 配對成功後，HC-05 的 LED 閃爍頻率會**變慢**（約每 2 秒閃一次）

### 6-3. 找到 COM 埠號

- 開始選單 → 搜尋「**裝置管理員**」→ 開啟
- 展開「**連接埠（COM 和 LPT）**」
- 找到 **標準串列藍牙連結（COM?）**，會有兩個：
  - 一個是「傳入（Incoming）」
  - 一個是「傳出（Outgoing）」
- **記下傳出的 COM 埠號**（例如 COM6），這是 Python 要用的

> 也可以：開始選單 → 搜尋「藍牙設定」→「更多藍牙設定」→「COM 連接埠」分頁查看

## 7. 用 Python 測試藍牙通訊

### 7-1. 安裝 Python 套件

```bash
pip install pyserial
```

### 7-2. 設定 COM 埠

開啟 `test_bt.py`，修改最上方兩個參數：

```python
BT_PORT  = 'COM4'   # ← 藍牙傳出 COM 埠（步驟 6-3 查到的）
USB_PORT = 'COM3'   # ← Arduino USB COM 埠（步驟 3-3 查到的）
```

### 7-3. 執行測試

確認 Arduino 已透過 USB 連接電腦、HC-05 已配對，然後執行：

```bash
python test_bt.py
```

### 7-4. 確認結果

若藍牙通訊正常，終端機會顯示：

```
✅ Arduino 有收到藍牙訊號！(IBUS 油門通道: 1502)
```

- 確認 HC-05 的 LED 是緩慢閃爍（已配對），而非快速閃爍（未配對）
- 確認 D10 接線沒有鬆脫

# Part 2：Arduino 與飛控板連接

## 11. 連接前注意事項

### 11-1. 接線方式 Disclaimer

> ⚠️ Arduino Nano 的腳位是標準 2.54mm 排針，但飛控板（F405）上通常是焊盤或小孔，**沒有辦法用一般杜邦線穩定插接**。

目前可行的暫時固定方式：

- **膠帶固定法**：將杜邦線母頭壓在飛控板焊盤上，用膠帶固定住
- **穿孔法**：將杜邦線母頭那端的銅線剪出來，把裸銅線穿過飛控板對應點位的小孔後彎折固定

若要長期穩定使用，建議將導線焊接到飛控板焊盤上。

## 12. 飛控板連接電腦與韌體更新

## 15. Arduino 與飛控板接線

### 15-1. 接線說明

此階段兩塊板子的電源皆由電腦 USB 供應，**不需要另外接電源線**，只需要接兩條線：

| 從哪裡 | 到哪裡 | 說明 |
|--------|--------|------|
| Arduino **D1（TX）** | 飛控板 **RX1** | IBUS 訊號線 |
| Arduino **GND** | 飛控板 **GND** | 共地線（必接，否則訊號不穩）|

> ⚠️ 注意：Arduino 的 IBUS 訊號是從硬體串列埠的 **D1（TX）** 送出，不是 D11。D11 是藍牙模組的 SoftwareSerial TX，兩者不同。

### 15-2. 確認接線

接好後，兩塊板子同時透過各自的 USB 連接電腦，確認：
- Arduino USB 連線正常（可透過 Arduino IDE 確認 COM 埠存在）
- 飛控板 USB 連線正常（Betaflight 可以 Connect）

### 15-3. 焊接永久連線

> ⏳ **建議等到 Part 3 全部完成（馬達轉向確認、手把控制馬達整合測試通過）之後再焊接。**  
> 焊接前仍可用跳線暫時連接繼續測試，確認整個系統運作正常再做永久固定，避免焊完才發現問題要拆掉重焊。

所有功能確認無誤後，將連線焊接固定。**建議使用 24 AWG 線材**，細度適合板上焊點，不易造成短路。

完整焊接接線如下：

| 從哪裡 | 到哪裡 | 說明 |
|--------|--------|------|
| Arduino **5V** | HC-05 **VCC** | 藍牙模組供電 |
| Arduino **5V** | 飛控板 **5V** | 飛控板供電（與藍牙共用同一 5V 點位）|
| Arduino **D1（TX）** | 飛控板 **RX1** | IBUS 訊號線 |
| Arduino **GND** | 飛控板 **GND** | 共地線 |
| Arduino **GND** | HC-05 **GND** | 藍牙模組共地（若尚未接）|

> ⚠️ Arduino 的 5V 點位同時供電給藍牙模組和飛控板，三點共用同一電源，焊接時確認接點牢固不虛焊。

## 16. Betaflight 安全設定

連線飛控板後，先進行重要的安全設定，**避免測試馬達時發生危險**。

### 16-1. 關閉 Airmode

Airmode 會在無人機解鎖狀態下持續對馬達輸出最低轉速，若在地面測試時開著槳葉，可能導致爆衝翻車。

1. 左側選單點選「**Configuration**」
2. 往下滑找到「**Other Features**」區塊
3. 確認 **`AIRMODE`** 為**關閉**狀態（灰色）
4. 點選右下角「**Save and Reboot**」

## 17. 通道對應與訊號確認

### 17-1. 設定通道對應（Channel Map）

PS 手把的搖桿軸向預設順序與飛控板期待的順序不同，若不修正會導致解鎖瞬間油門通道被誤判造成爆轉。

1. 左側選單點選「**Receiver**」
2. 找到「**Channel Map**」，將其設定為 **`AETR1234`**
3. 點選右下角「**Save and Reboot**」

### 17-2. 確認訊號是否正常

1. 執行 `main_althold_v2.py`
2. 回到 Betaflight **Receiver** 分頁
3. 不動搖桿時，確認 **Throttle 通道穩定停在 `1000`**
4. 移動各搖桿，確認對應通道數值有正確變化

✅ 數值正常移動 → IBUS 訊號通訊正常
❌ 數值全部靜止 → 回頭確認 Step 13 的 UART 設定與 Step 15 的接線

## 18. 模式綁定

進入「**Modes**」分頁，將控制通道綁定到對應功能。

### 18-1. ARM（解鎖）

程式預設 `arm_state = 0`，對應 AUX 1 送出 **1000**（上鎖）；按下 △ 鍵後切換為 **2000**（解鎖）。

滑桿設定邏輯：
- 滑桿範圍必須**只涵蓋 2000**，不能涵蓋 1000
- 建議設定範圍：**1800 ~ 2100**

```
AUX 1 數值：
  1000 ──────────────────│     │──── 2000
  （上鎖，預設）         不啟動  啟動（解鎖）
                        1800  2100
```

這樣一來，程式啟動時預設送 1000，飛控板判斷為上鎖狀態；按 △ 鍵送出 2000 才會解鎖。

設定步驟：
1. 找到 **ARM** 那列 → 點選「**Add Range**」
2. 通道選 **AUX 1**
3. 將滑桿兩端拖曳到 **1800 ~ 2100** 的範圍
4. 點選「**Save**」

### 18-2. ANGLE（自穩模式）

程式在 Arduino sketch 中將 CH6（AUX 2）固定輸出 **2000**，代表自穩模式永遠啟動。

設定步驟：
1. 找到 **ANGLE** 那列 → 點選「**Add Range**」
2. 通道選 **AUX 2**
3. 滑桿範圍同樣設在 **1800 ~ 2100**（涵蓋固定輸出的 2000）
4. 點選「**Save**」

---

## 19. 解鎖失敗排查

按 △ 鍵（或 R 鍵）無法解鎖時，進入「**Setup**」分頁，查看右側的 **Arming Disable Flags**：

| Flag | 原因 | 解決方式 |
|------|------|----------|
| `MSP` | 飛控板仍連著 Betaflight | 拔掉 USB，斷開 Betaflight 連線後再試 |
| `RX_FAILSAFE` | 訊號中斷 | 拔掉電池與 USB，重新插上讓 Arduino 與飛控板同步啟動 |
| `THROTTLE` | 油門未歸零 | 按 □ 鍵重新校準搖桿，確認送出的油門訊號為 1000 |
| `ARM_SWITCH` | 解鎖開關未復位 | 確認程式內 `arm_state` 為 0（上鎖狀態）再重新解鎖 |

# Part 3：電調、馬達與電源

## 20. 焊接安全警告

> 🔥 **焊接具有危險性，操作前請確保以下條件：**
>
> - 在**通風良好**的環境進行，焊錫煙霧有害健康
> - 烙鐵溫度極高（300°C 以上），**不得觸碰烙鐵頭**，用完立刻放回架上
> - 工作台周圍**不得有易燃物**（紙張、布料、塑膠袋）
> - 建議佩戴護目鏡，防止焊錫噴濺
> - 焊接完成後確認烙鐵已**完全冷卻**再收納

---

## 21. 電調與電源線焊接

### 21-1. 確認電源線極性

電源線焊接前**必須確認正負極**，接反會立即燒毀電調與飛控板。

| 線色 | 極性 |
|------|------|
| 紅色 | 正極（+） |
| 黑色 | 負極（−） |

使用三用電表的「直流電壓」檔，將紅棒接電池正極、黑棒接負極，確認讀值為正數後再對應焊接。

### 21-2. 確認電容極性

電容焊接錯誤會導致爆炸，**務必在焊接前確認方向**：

- 電容本體上有白色或灰色條紋的那側為**負極（−）**
- 較長的引腳為**正極（+）**
- 焊接時正極接電源正極焊盤，負極接地

> ⚠️ 電解電容接反後通電會膨脹甚至爆炸，後果嚴重，請仔細確認。

### 21-3. 焊接順序建議

1. 先將電容焊到電調的電源輸入焊盤（靠近 XT60 接頭處）
2. 再將電源線（XT60 接頭）焊到電調電源輸入焊盤
3. 確認每個焊點光滑飽滿，無虛焊或冷焊

---

## 22. 電調與馬達焊接

### 22-1. 無刷馬達接線原理

無刷馬達有 **3 條相線**（通常為黑/白/紅，或 A/B/C），電調也有對應的 **3 個輸出焊盤**。三條線的接法決定馬達轉向：

- 三條線**依序對接**（A→A、B→B、C→C）：馬達以某個方向旋轉
- 將**任意兩條線對調**：馬達反轉

> 四軸無人機的四顆馬達有固定的轉向規律（對角線同向），先全部接好再用 Betaflight 確認，若轉向錯誤再對調兩條線。

### 22-2. 焊接步驟

1. 將馬達三條線末端的絕緣皮剝除約 3mm，用烙鐵沾錫上錫（鍍錫）
2. 在電調對應馬達的三個輸出焊盤上先點上錫
3. 將馬達線依序焊到電調焊盤，**確保每個焊點牢固**
4. 輕拉導線確認不會脫落
5. 重複以上步驟完成四顆馬達

### 22-3. 對應關係

| 電調輸出 | 對應馬達 | Betaflight 編號 |
|----------|----------|-----------------|
| M1 | 馬達 1（右前） | Motor 1 |
| M2 | 馬達 2（左後） | Motor 2 |
| M3 | 馬達 3（左前） | Motor 3 |
| M4 | 馬達 4（右後） | Motor 4 |

> 實際位置依機架設計可能不同，以 Betaflight Setup 頁面顯示的圖示為準。

## 23. 上電安全檢查

### 23-1. 接上電池前確認

- 所有焊點目視檢查，確認無短路、無金屬碎屑殘留
- 電源線正負極再次確認
- 馬達線確認無裸露銅線互相接觸

### 23-2. 第一次接上電池

> ⚠️ 第一次通電時，雙眼**專注觀察飛控板與電調**，手保持在電池接頭旁，一旦發現異常立刻斷電。

接上電池後的正常現象：
- 飛控板 LED 亮起
- 電調發出 「嗶嗶嗶」啟動音（馬達數）
- **無冒煙、無異味、無過熱**

若發現冒煙或異味：
1. **立刻拔除電池**
2. 靜置待冷卻，不要立即觸碰
3. 目視檢查是否有元件燒焦或變色
4. 找出問題焊點，重新焊接後再次通電測試

## 24. 用 Betaflight 確認馬達連接

通電且無異常後，透過 Betaflight 的馬達測試頁面確認每顆馬達是否正確連接。

> ⚠️ **測試前請拆除所有槳葉**，馬達轉動時槳葉會造成嚴重傷害。
>
> 💡 若尚未焊接全部四顆馬達，**先接一顆馬達測試即可**，確認電調與 Betaflight 通訊正常後再繼續焊接其餘馬達。
>
> 📌 **此步驟不需要連接 Arduino**，只需飛控板透過 USB 連接電腦，並接上電池供電即可。

### 24-1. 進入馬達測試頁面

1. 用 USB 連接飛控板到電腦（電池同時接著供電）
2. 開啟 Betaflight → 左側選單點選「**Motors**」
3. 勾選頁面上的安全確認勾選框（「I understand the risks...」）

### 24-2. 逐一測試馬達

1. 將 **Master** 滑桿維持在 0
2. 拖曳 **Motor 1** 滑桿到約 1100，觀察對應位置的馬達是否轉動
3. 確認轉動後將滑桿歸零
4. 依序測試 Motor 2、3、4

✅ 馬達可正常驅動 → 電調與馬達焊接正常
❌ 馬達無反應 → 檢查該馬達的焊點與電調輸出是否正確

### 24-3. 確認轉向

對照 Betaflight **Setup** 頁面左上角的馬達位置圖，確認每顆馬達的轉向：

```
      前
  CW  ↑  CCW
  (2)   (1)
    ╲   ╱
    ╱   ╲
  (3)   (4)
  CCW      CW
```

> 對角線的兩顆馬達轉向相同；相鄰的馬達轉向相反。

### 24-4. 用 Betaflight 反轉馬達方向

若某顆馬達轉向錯誤，可用 Betaflight 軟體直接反轉，**不需要重新焊接**：

1. 確保**已連接電池**（ESC 需供電才能接收設定指令）
2. Betaflight → 左側選單點選「**Motors**」
3. 在頁面下方找到每顆馬達旁的 **「Motor Direction」** 下拉選單
   - `Normal` = 預設方向
   - `Reversed` = 反轉
4. 將轉向錯誤的馬達改為 `Reversed`，點選 **「Save」**
5. 重新進行 24-2 的滑桿測試，確認方向已修正

> 💡 若你的 Betaflight 版本較舊，Motors 頁面可能沒有 Motor Direction 選項。此時需改用 **BLHeli Configurator**（獨立軟體）連接 ESC 來反轉，或直接將該馬達的**任意兩條相線對調**後重新焊接。

## 25. 手把控制馬達整合測試

完成馬達單獨測試後，將飛控板與 Arduino 接上，確認能透過手把（或鍵盤）實際驅動馬達。

> ⚠️ **測試前請拆除所有槳葉。**
>
> ⚠️ 進行此測試前，請再次確認：
> - Betaflight **AIRMODE 已關閉**（Step 16-1）
> - Betaflight Motors 頁面的**安全確認勾選框已勾選**，否則手把解鎖後馬達不會轉動

### 25-1. 連接所有硬體

1. Arduino 透過 USB 連接電腦
2. 飛控板透過 USB 連接電腦
3. 接上電池（供電給電調與馬達）
4. 確認 Arduino D1（TX）已接到飛控板 RX1，GND 共地

### 25-2. 開啟 Betaflight Motors 頁面

1. 開啟 Betaflight，連線飛控板
2. 左側選單點選「**Motors**」
3. 確認安全確認勾選框**已勾選**

### 25-3. 執行控制程式

開啟另一個終端機，執行主程式：

```bash
python main_althold_v2.py
```

有插手把會自動進入手把模式，沒有則自動進入鍵盤模式。

### 25-4. 測試流程

1. 確認油門歸零（搖桿置中或 Tab/Shift 未按）
2. 按 **△ 鍵**（或 **R 鍵**）解鎖
3. 緩慢推動油門，觀察 Betaflight Motors 頁面的數值是否隨之變化
4. 馬達應開始轉動

✅ 馬達隨搖桿輸入轉動 → 整合測試通過，系統正常
❌ 馬達無反應 → 依序確認：
  - Betaflight Receiver 頁面訊號是否正常（Step 17-2）
  - ARM 模式是否正確解鎖（Step 19 解鎖失敗排查）
  - Motors 頁面安全勾選框是否已勾
