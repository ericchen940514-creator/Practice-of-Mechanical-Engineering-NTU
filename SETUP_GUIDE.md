## 1. 所需材料

| 材料 | 數量 | 備註 |
|------|------|------|
| Arduino Nano | 1 | ATmega328P，建議附 USB 傳輸線 |
| HC-05 藍牙模組 | 1 | 注意：HC-05 可做主從兩用；HC-06 只能做從機 |
| 麵包板 | 1 | — |
| 杜邦線（母對母） | 若干 | Arduino Nano 與 HC-05 皆預設已焊排針 |
| 電阻 1kΩ | 1 | 電壓分壓用 |
| 電阻 2kΩ | 1 | 電壓分壓用（可用兩個 1kΩ 串聯代替）|
| Windows 電腦 | 1 | 需有內建藍牙或藍牙 USB 接收器 |

## 2. 安裝 Arduino IDE

1. 前往官網下載：https://www.arduino.cc/en/software
2. 選擇 **Windows Win 10 and newer, 64 bits**，下載安裝檔
3. 安裝過程全部按「下一步」即可，**不要取消勾選驅動程式安裝**

> Arduino IDE 會自動安裝 USB 驅動程式（CH340 或 FTDI），讓電腦認得 Arduino Nano。

## 3. 驗證 Arduino Nano 是否正常

這步用內建的 Blink 範例，確認板子通電正常、燒錄成功。

### 3-1. 用 USB 連接 Arduino Nano 到電腦

### 3-2. 開啟 Arduino IDE，設定板子

- 上方選單 → **工具（Tools）**
- **板子（Board）** → 搜尋並選 **Arduino Nano**
- **處理器（Processor）** → 先選 **ATmega328P（Old Bootloader）**

> 若燒錄失敗，再改成 **ATmega328P**（不加 Old Bootloader）試試，兩種 bootloader 版本都很常見。

### 3-3. 選擇正確的 COM 埠

- **工具 → 連接埠（Port）** → 選擇帶有「USB」或「CH340」字樣的 COM 埠
- 若不確定是哪個，先拔掉 Arduino，看哪個 COM 埠消失，那個就是

### 3-4. 上傳 Blink 範例

- **檔案（File）→ 範例（Examples）→ 01.Basics → Blink**
- 點選上傳按鈕（→ 箭頭圖示）
- 等待下方顯示「**上傳完成（Done uploading）**」

### 3-5. 確認結果

Arduino Nano 板子上的 **L 燈（位於板子中央）** 應該開始每秒閃爍一次。

✅ **閃爍正常 → 板子沒問題，繼續下一步**
❌ **上傳失敗 → 改換處理器版本（3-2 步驟），或換一條 USB 線**

## 4. HC-05 藍牙模組接線

### 4-1. 認識 HC-05 的腳位

HC-05 模組通常有 6 支腳，從左到右（依模組正面標示）：

```
[ STATE | RXD | TXD | GND | VCC | EN ]
```

| 腳位 | 說明 |
|------|------|
| STATE | 連線狀態指示（可不接） |
| RXD | 接收資料（模組的輸入，3.3V 邏輯）|
| TXD | 傳送資料（模組的輸出）|
| GND | 接地 |
| VCC | 電源 3.3V ~ 5V |
| EN | 進入 AT 指令模式用（一般使用時不接）|

### 4-2. 接線圖

本專題的資料流向是**單向的**：電腦發送指令 → HC-05 → Arduino，Arduino 不需要將資料回傳給電腦。因此：

- **D10（Arduino RX ← HC-05 TXD）**：必須接，這條是接收指令用的
- **D11（Arduino TX → HC-05 RXD）**：**目前可以不接**，因為程式不需要從 Arduino 回傳資料

> 📝 **D11 留待未來擴充用**（例如：回傳感測器數值、飛行狀態到電腦）。若日後需要接上，HC-05 的 RXD 只接受 3.3V 邏輯電位，而 Arduino TX 輸出為 5V，**需透過電阻分壓降壓後再接**（見下方說明）。

#### 目前所需接線（3 條）

```
Arduino Nano                    HC-05
─────────────                   ─────
    5V  ──────────────────────  VCC
   GND  ──────────────────────  GND
   D10  ──────────────────────  TXD   （接收電腦指令）
   D11  ✗ 暫不接               RXD   （未來回傳資料用）
```

| 連接方式 | 從哪裡 | 到哪裡 |
|----------|--------|--------|
| 母對母杜邦線 | Arduino **5V** | HC-05 **VCC** |
| 母對母杜邦線 | Arduino **GND** | HC-05 **GND** |
| 母對母杜邦線 | Arduino **D10** | HC-05 **TXD** |

#### 日後擴充：D11 → HC-05 RXD 的降壓接法

若未來需要 Arduino 回傳資料，D11 不能直接接到 HC-05 RXD，需要加電阻分壓：

```
D11 (5V) ─── 1kΩ ─── 節點 ─── 2kΩ ─── GND
                       ↑
                  HC-05 RXD
                 ≈ 3.33V ✅
```

5V × 2kΩ ÷ (1kΩ + 2kΩ) ≈ **3.33V**，安全範圍內。電阻部分需借助麵包板串接。

### 4-3. 接線完成後確認

- HC-05 模組上有一顆 **LED 燈**
- 通電後應快速閃爍（約每秒 2 次）→ 代表等待配對中，**接線正確**
- 若完全不亮 → 檢查 VCC / GND 是否接對

### 4-4. （選用）更改藍牙模組名稱與密碼

HC-05 出廠預設名稱為 `HC-05`、配對密碼為 `1234`。若需要更改，需透過 **AT 指令模式**設定，步驟如下：

#### 步驟一：上傳 AT 中繼程式

先拔掉 HC-05 的 D10 接線，然後上傳以下程式（讓電腦透過 Arduino 跟 HC-05 溝通）：

```cpp
#include <SoftwareSerial.h>
SoftwareSerial BTSerial(10, 11);

void setup() {
  Serial.begin(38400);    // AT 模式固定用 38400
  BTSerial.begin(38400);
  Serial.println("AT 模式已就緒，請輸入指令...");
}

void loop() {
  if (BTSerial.available()) Serial.write(BTSerial.read());
  if (Serial.available())   BTSerial.write(Serial.read());
}
```

#### 步驟二：讓 HC-05 進入 AT 模式

1. **先拔掉 HC-05 的電源（VCC 線）**
2. 用手按住 HC-05 模組上的小按鈕（EN 按鈕）不放
3. 接回 VCC，繼續按住 2 秒後放開
4. HC-05 的 LED 改為**緩慢閃爍（約每 2 秒一次）**→ 代表已進入 AT 模式

> 若 HC-05 沒有按鈕，可將 EN 腳接到 3.3V 後再通電，效果相同。

#### 步驟三：用 Serial Monitor 發送指令

- Arduino IDE → **工具 → 序列埠監控視窗（Serial Monitor）**
- 右下角設定：鮑率 **38400**、換行選 **Both NL & CR**
- 輸入 `AT` 後按 Enter，若回應 `OK` 代表 AT 模式正常

#### 常用 AT 指令

| 指令 | 說明 | 範例 |
|------|------|------|
| `AT` | 測試連線 | 回應 `OK` |
| `AT+NAME=新名稱` | 更改藍牙名稱 | `AT+NAME=Drone_BT` |
| `AT+PSWD=新密碼` | 更改配對密碼 | `AT+PSWD=9999` |
| `AT+NAME?` | 查詢目前名稱 | 回應 `+NAME:HC-05` |
| `AT+PSWD?` | 查詢目前密碼 | 回應 `+PSWD:1234` |

**範例操作流程：**

```
AT
OK

AT+NAME?
+NAME:HC-05
OK

AT+NAME=Drone_BT
OKName:Drone_BT

AT+PSWD=9999
OK
```

#### 步驟四：設定完成後恢復正常模式

斷電再重新接上 VCC（**不要按** EN 按鈕），LED 恢復快速閃爍即代表已回到正常模式。

接著繼續 Step 5，將正式程式燒錄回 Arduino。

## 5. 燒錄正式程式到 Arduino

**先拔掉 HC-05 的 TXD 接線（D10）**，再連接 USB 燒錄。

> 原因：燒錄時若串列腳位有其他裝置連接，可能導致燒錄失敗或衝突。

### 5-1. 開啟 `sketch_mar26a.ino` 並上傳

- Arduino IDE → **檔案 → 開啟** → 選擇 `sketch_mar26a/sketch_mar26a.ino`
- 確認板子、處理器、COM 埠設定與第 3 步相同
- 點選上傳，等待「**上傳完成（Done uploading）**」

上傳完成後，**再把 D10 的接線接回去**。

## 6. 電腦配對 HC-05

### 6-1. 開啟 Windows 藍牙設定

- 開始選單 → 設定 → **藍牙與裝置** → 開啟藍牙 → **新增裝置**

### 6-2. 配對 HC-05

- 掃描到名稱為 **HC-05**（或 **H-C-2010**）的裝置 → 點選
- 輸入配對碼：**1234**（預設值，若失敗試 **0000**）
- 配對成功後，HC-05 的 LED 閃爍頻率會**變慢**（約每 2 秒閃一次）

### 6-3. 找到 COM 埠號

- 開始選單 → 搜尋「**裝置管理員**」→ 開啟
- 展開「**連接埠（COM 和 LPT）**」
- 找到 **標準串列藍牙連結（COM?）**，會有兩個：
  - 一個是「傳入（Incoming）」
  - 一個是「傳出（Outgoing）」
- **記下傳出的 COM 埠號**（例如 COM6），這是 Python 要用的

> 也可以：開始選單 → 搜尋「藍牙設定」→「更多藍牙設定」→「COM 連接埠」分頁查看

## 7. 用 Python 測試藍牙通訊

### 7-1. 安裝 Python 套件

```bash
pip install pyserial
```

### 7-2. 設定 COM 埠

開啟 `test_bt.py`，修改最上方兩個參數：

```python
BT_PORT  = 'COM4'   # ← 藍牙傳出 COM 埠（步驟 6-3 查到的）
USB_PORT = 'COM3'   # ← Arduino USB COM 埠（步驟 3-3 查到的）
```

### 7-3. 執行測試

確認 Arduino 已透過 USB 連接電腦、HC-05 已配對，然後執行：

```bash
python test_bt.py
```

### 7-4. 確認結果

若藍牙通訊正常，終端機會顯示：

```
✅ Arduino 有收到藍牙訊號！(IBUS 油門通道: 1502)
```

若一直顯示 `⏳ 等待確認...`：
- 確認藍牙 COM 埠號是否正確（裝置管理員查看）
- 確認 HC-05 的 LED 是緩慢閃爍（已配對），而非快速閃爍（未配對）
- 確認 D10 接線沒有鬆脫

---

完成以上步驟後，你已經驗證了：

- ✅ Arduino Nano 板子功能正常
- ✅ HC-05 接線正確、可以通電
- ✅ 電腦可以透過藍牙將指令送進 Arduino

### 8-2. 將手把透過藍牙與電腦配對

### 8-3. 執行按鍵編號測試

```bash

```

終端機會顯示偵測到的手把名稱，接著按下手把上的每一個按鍵，確認畫面有出現對應的編號：

```
🎮 偵測到手把: Wireless Controller
👉 現在請按下 L1 和 R1，看看畫面上顯示 Button 幾號...
```

### 8-4. 確認按鍵編號與程式一致

對照 `controller.py` 最上方的按鍵定義，確認編號相符：

```python
BTN_SQUARE = 2
BTN_TRI    = 3
BTN_L1     = 9
BTN_R1     = 10
BTN_UP, BTN_DOWN, BTN_LEFT, BTN_RIGHT = 11, 12, 13, 14
BTN_OPTIONS = 6
```

若編號不同（例如換了不同廠牌手把），直接修改 `controller.py` 對應的數字即可。

## 10. （備用）無手把鍵盤控制模式

沒有手把時，可以改用 `keyboard_controller.py` 透過鍵盤操控。

### 按鍵對照表

| 按鍵 | 對應功能 |
|------|----------|
| `W` / `S` | 左搖桿 上/下（油門增減） |
| `A` / `D` | 左搖桿 左/右（偏航）|
| `↑` / `↓` | 右搖桿 上/下（俯仰）|
| `←` / `→` | 右搖桿 左/右（翻滾）|
| `Tab` / `Shift` | 基準油門 上/下 |
| `C` / `Z` | 每次調整步進量 增大/縮小 |
| `Q` / `E` | 夾爪 閉合/張開 |
| `R` | 解鎖 / 上鎖（油門須歸零才能解鎖）|
| `X` 長按 3 秒 | 安全退出 |

### 執行方式

確認 `keyboard_controller.py` 最上方的 COM 埠設定後執行：

```bash
python keyboard_controller.py
```

執行後會彈出一個小視窗，**必須點選該視窗**讓它取得焦點，鍵盤輸入才會被偵測到。
