## 與定高模式（v2/v5）的根本差異

| | 定高模式（v2/v5） | **速度模式（v3/v6）** |
|-|------------------|----------------------|
| Arduino PID 目標 | 位置（target_alt_cm） | **速度（vel_cmd cm/s）** |
| 外環 | 位置 P | 無 |
| 搖桿功能 | 更新目標高度 | 直接送速度命令 |
| 放開搖桿 | 主動拉回目標高度 | vel_cmd = 0，積分維持懸停 |
| 飛上平台 | 感測器誤讀 → 位置外環暴衝 | 無位置外環，不受影響 |

## 系統架構

### 通訊總迴圈

```
【Python 地面站】                                【Arduino（無人機端）】
  ├─ 1. 讀手把輸入                               ├─ 1. 讀 VL53L0X（每 20ms）
  │    手動模式：套 Expo 曲線                    │    ├─ EMA 低通（ALPHA=0.35）
  │    定速模式：線性（無 Expo）                 │    ├─ Outlier rejection（±200mm）
  ├─ 2. 計算速度命令                             │    └─ Freeze 確認 → 重置 PID 狀態 + 跳過 EMA
  │    vel_cmd = stick × 60 cm/s                ├─ 2. 解析封包，更新 received_vel_cmd
  │    alt_byte = vel_cmd + 128                 ├─ 3. 速度 PID（每 20ms）
  ├─ 3. 打包送出                                 ├─ 4. IBUS 輸出（50Hz）
  │  ──────────── Bluetooth ─────────────>     └─ 5. 退出時送 T: 同步油門
  │<─── D:<mm> / P:<ibus>（每 200ms）──────────
  └─ 4. UI 顯示高度與速度命令
```

### Arduino 速度 PID 架構

```
Python 搖桿
    │  × ALT_VEL_SCALE (60 cm/s)
    ▼
vel_cmd（±60 cm/s 夾限）
    │  超過 MAX_ALT_CM (120cm) → 強制 ≤ −5 cm/s
    ▼
速度誤差 = vel_cmd − v_est
    │
    ▼  Kp_vel=0.80  Ki_vel=1.20  Kd_vel=0.0
油門偏移（±220 IBUS 夾限，速率限制 ±20 IBUS/20ms）
    │
    ▼
hover_estimate + 偏移 → IBUS（1100～1750）→ 飛控
```

速度估計：`v_est = GAMMA×(Δcm/Δt) + (1−GAMMA)×v_est`（GAMMA=0.20）

### 封包格式（10 bytes）

```
S | thr | yaw | pitch | roll | alt_byte | gripper | arm | ah_val | XOR
```

| 欄位 | 說明 |
|------|------|
| `thr` | 手動基準油門（0～255）；定速時 Arduino 不使用此值 |
| `alt_byte` | 速度命令：`vel_cm_s + 128`，128=0 cm/s，>128=上升，<128=下降 |
| `ah_val` | 0=手動，1=定速，2=積分重設 |

### Arduino → Python 回傳

| 格式 | 說明 |
|------|------|
| `D:<mm>` | 每 200ms，EMA 濾波後高度 |
| `P:<ibus>` | 每 200ms（定速時），PID 輸出油門 |
| `T:<ibus>` | 退出定速時一次，Python 換算後同步基準油門 |

## 操作說明（手把模式）

| 按鍵 | 功能 |
|------|------|
| **△** | 解鎖 / 上鎖 |
| **○** | 切換定速 開 / 關 |
| **□** | 定速時：積分重設（重新收斂懸停油門） |
| **按鍵 15** | 右搖桿偏移補正（手動模式，搖桿推到想補正位置時按） |
| **L1 / R1** | 夾爪閉合 / 張開 |
| **D-pad 上/下** | 手動模式：基準油門 ±步進 |
| **D-pad 左/右** | 手動模式：油門步進量 ±1 |
| **Options 長按 3 秒** | 安全退出 |
| **按鍵 4 + 6 同時** | 🚨 緊急停機 |

**左搖桿（定速模式中）：**

| 動作 | vel_cmd | 效果 |
|------|---------|------|
| 中心（死區內） | 0 cm/s | 積分維持當前懸停高度 |
| 推上 | 正值（最大 60 cm/s） | 爬升 |
| 推下 | 負值（最大 −60 cm/s） | 下降 |

## 建議測試流程

| 步驟 | 操作 |
|------|------|
| 1 | 確認 VL53L0X 接線（**3.3V，非 5V**） |
| 2 | 燒錄 `sketch_althold_v6_velmode.ino` |
| 3 | 執行 `main_althold_v3_velmode.py`，確認 UI 顯示高度有數值 |
| 4 | 解鎖，手動飛到穩定懸停高度（約 15～30cm） |
| 5 | 按 ○ 切入定速，放開左搖桿，觀察是否懸停 |
| 6 | 推左搖桿上/下，確認高度跟隨搖桿變化 |
| 7 | 按 ○ 退出定速，手動降落 |

> 從地面直接啟動定速亦可，但 `hover_estimate` 需要數秒積分收斂，期間高度可能緩慢漂移

> 若震盪劇烈立刻按 ○ 切回手動，或 **4+6 緊急停機**

## 感測器 Freeze 恢復機制

VL53L0X 連續 5 次回傳與前次差距 >200mm 時，視為高度已真實大幅改變（例如飛上平台），此時：

1. `filtered_mm` 直接換成最新原始讀值（跳過 EMA 混合）
2. 全部 PID 狀態清零（`integral_vel`、`last_vel_error`、`v_est`、`prev_cm_filt/raw`）
3. 通知 Python 送出 `F:1`

這樣可防止舊積分在高度突變後造成暴衝。

## PID 數據記錄（pid_logger）

- 記錄條件：**解鎖（arm）且定速模式同時開啟**才會寫入
- 每次「上鎖後重新解鎖 + 開定速」都會產生新的 CSV 檔（時間戳命名）
- 儲存於 `vel_logs/` 資料夾，欄位：`time_s, current_alt_cm, target_alt_cm, vel_cmd_cm_s, pid_thr_ibus, pid_thr_0_255`

## 已知限制

- 純速度 PID：無位置外環，放開搖桿只靠積分維持，外力擾動（陣風）不會被主動修正
- `hover_estimate` 初始值若偏低（從地面直接啟動），積分需 2～5 秒才收斂
- 超過 120cm 觸發強制 −5 cm/s 安全下降，加上慣性實際可能飛到 ~130cm 才回頭
- VL53L0X 飛行傾斜時讀值偏差（測量的不是真正垂直距離）
