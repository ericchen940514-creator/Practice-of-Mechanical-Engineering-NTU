# 定高 PID 調整指南

## 目前參數（v0.4.15.0）

```cpp
const float Kp    = 2.5;
const float Ki    = 0.5;
const float Kd    = 3.5;
integral = constrain(integral, -300.0f, 300.0f);  // 最多 ±150 IBUS 補償
```

---

## 調參歷史

### 原始版本（git 提交版，April 16）
```
Kp=2.5  Ki=0.3  Kd=0.8  積分 ±150
```
問題：Ki 太小（最多 ±45 IBUS），base_throttle 設錯導致無人機根本無法起飛，整個 April 16 的 log 都在地面。

### 第一次調整
```
Kp=4.0  Ki=0.5  Kd=1.5  積分 ±300
```
原因：Kp 加大增加爬升力，Ki 提高補償電池衰減，積分上限擴大。  
結果：**第 11 次測試（althold_20260417_162654）出現發散震盪。**

### 第 11 次測試為何發散（重要）

目標 24cm，無人機在 24cm 附近飛行，高度誤差 ±2cm 逐漸擴大到 ±10cm，最終發散。

**根本原因：Kp=4.0 過大。**

以振盪週期 ~0.5 秒估算：高度以 ±3cm 振盪時，速度最大約 37 cm/s。Kd 的修正量 = Kd × 速度 = 1.5 × 37 = 55 IBUS。而 Kp 的修正量 = Kp × 誤差 = 4.0 × 3 = 12 IBUS。**Kp 在誤差縮小時油門驟降，Kd 又不夠壓住速度，導致每個週期誤差越來越大。**

Ki 不是原因——Ki 影響低頻補償，不會引起 0.5 秒週期的快速振盪。

### 第二次調整（現在）
```
Kp=2.5  Ki=0.5  Kd=3.5  積分 ±300
```
- Kp 降回 2.5：回到不會發散的保守值
- Kd 大幅提高至 3.5：補足 Kp 減小後的阻尼不足
- Ki 維持 0.5：補償電池衰減造成的穩態誤差

## 安全保護（v0.4.15.0 新增）

```cpp
const float MAX_ALT_CM      = 60.0f;   // 超過此高度強制解除定高
const float MAX_CORRECTION  = 150.0f;  // PID 單次修正量上限（IBUS）
```

- **修正量上限**：PID 輸出最多偏離 base_throttle ±150 IBUS，防止暴衝
- **高度上限**：超過 60cm 自動解除定高回手動
- **起飛逾時**：8 秒內未離地自動解除

---

## 起飛流程（v0.4.15.0 起）

1. 解鎖無人機
2. 按 ○ 切入定高（預設目標 25cm）
3. Arduino 自動緩坡爬升（每 20ms +2 IBUS）
4. 高度超過 15cm 後快照懸停油門，切換為 PID
5. PID 繼續爬升至目標高度

> 不需要手動調油門到懸停值，系統自動偵測。

## 測試流程

| 步驟 | 操作 | 等待時間 |
|------|------|---------|
| 1 | 解鎖，按 ○ 切入定高（目標 25cm） | — |
| 2 | 觀察爬升與穩定 | 15 秒 |
| 3 | D-pad 上，目標 +10cm（→35cm） | 15 秒 |
| 4 | D-pad 下，目標 -10cm（→25cm） | 10 秒 |
| 5 | 按 ○ 關閉定高，降落 | — |

## 診斷對照表

| 現象 | 診斷 | 建議調整 |
|------|------|---------|
| 爬升後一直低於目標 | Ki 太小 或 積分上限不足 | 加大 Ki 或積分上限 |
| 到目標後繼續衝過去 | Kp 太大 | 降低 Kp，加大 Kd |
| 高度震盪且越來越大（發散） | Kp 過大 | 降低 Kp |
| 高度震盪但不發散 | Kd 不足 | 加大 Kd |
| 目標改變後反應很慢 | Kp 太小 | 加大 Kp |
| 油門曲線高頻跳動 | 感測器噪訊放大 | 降低 Kd 或加大 ALPHA |
